Thiết kế v0.3.0 · 2026-07-18

Workflow mới: phát hiện reconnect + duyệt trước khi sync

Giữ nguyên backup 1 chiều (v1) · Sync 2 chiều origin ↔ working có bước duyệt · Bỏ cloud (rclone) · Luôn giữ bản cũ để time-travel

1Bức tranh tổng thể

Một thư mục Bản gốc (origin), hai dòng chảy: backup 1 chiều như cũ, và bản làm việc 2 chiều trên USB/ổ mạng. Cloud bị gỡ hẳn.

2Watcher kết nối — poll rẻ, chỉ sờ thư mục gốc

Thread nền mỗi 5 giây gọi read_dir trên thư mục gốc của origin và working (1 syscall, không đi vào cây thư mục). Việc quét thay đổi file chỉ chạy khi thật sự cần sync.

3Luồng reconnect: luôn duyệt trước khi sync

Khi ổ working cắm lại / ổ mạng nối lại: app không tự sync. Nó tính plan, bật cửa sổ và chờ bạn duyệt.

Quy tắc bất biến: engine cất bản cũ vào version store trước mọi thao tác ghi đè/xóa (bước ⑥). Duyệt nhầm vẫn Hoàn tác được nguyên lần sync qua run_id, hoặc khôi phục từng file từ lịch sử.

4Định kỳ: tự chạy phần an toàn, conflict thì hỏi

Vòng định kỳ (mặc định 30 phút) giữ như cũ, thêm 2 luật mới: bỏ qua cặp đang mất kết nối, và conflict thì bật cửa sổ chờ duyệt thay vì im lặng.

5Chốt an toàn: rút USB ≠ xóa file

Lỗ hổng ở 0.2.0: scan_dir trả về chỉ mục rỗng khi thư mục không tồn tại. Nếu ổ working bị rút đúng lúc chạy định kỳ, diff hiểu là "mọi file đã bị xóa ở working" → lên kế hoạch xóa lan sang bản gốc.

6Time travel — lưới an toàn cuối cùng

Không thao tác nào phá hủy dữ liệu ngay lập tức. Mọi ghi đè/xóa đều đi qua version store (định danh theo hash BLAKE3) kèm journal theo run_id:

  • Hoàn tác cả lần sync: nút「Hoàn tác lần đồng bộ gần nhất」→ khôi phục nội dung file và baseline.
  • Khôi phục từng file (UI mới): nhập đường dẫn tương đối → xem danh sách phiên bản (thời gian · thao tác · kích thước) → khôi phục ra vị trí bạn chọn.
  • File bị xóa khi sync thực chất chỉ chuyển vào version store — như thùng rác có lịch sử.

7Những gì bị gỡ bỏ (cloud)

  • Code: sync/cloud.rs, field targets/rclone, các command v2_status, v2_add_target, v2_remove_target, v2_replicate, v2_connect_remote.
  • UI: section trạng thái rclone, kết nối Google Drive/OneDrive, danh sách đích cloud, nút「Đẩy cloud」.
  • Config cũ trên máy có sẵn targets vẫn đọc bình thường (field lạ bị bỏ qua khi parse).
Checklist duyệt thiết kế: ① reconnect luôn qua thẻ duyệt — đúng ý bạn? ② định kỳ tự chạy phần an toàn, chỉ hỏi khi conflict? ③ poll 5 giây chỉ read_dir gốc — đủ nhanh? ④ chốt an toàn chặn cả plan lẫn apply khi mất kết nối? ⑤ cloud gỡ sạch không để sót UI?
